## Публикация постов

### Одиночный бонус

#### `/publish_single [ID]`
Публикация одного бонуса по его ID.

**Два способа использования:**

1. **С указанием ID в команде:**
```
/publish_single bonus_123
```

2. **Интерактивный режим:**
```
/publish_single
→ Бот попросит ввести ID
→ Введите: bonus_123
→ Появится превью поста
→ Нажмите "✅ Опубликовать" или "❌ Отмена"
```

**Что происходит:**
- ✅ Бот находит бонус в базе данных
- ✅ Создает красиво оформленный пост
- ✅ Показывает превью для проверки
- ✅ После подтверждения публикует в канал
- ✅ Сохраняет в историю публикаций

**Возможные ошибки:**
- ❌ "Бонус не найден" - проверьте ID
- ⚠️ "Достигнут лимит постов" - превышен дневной лимит (по умолчанию 4)

---

### Подборка из 3 бонусов

#### `/publish_bundle [ID1,ID2,ID3]`
Публикация подборки из трёх бонусов.

**Два способа использования:**

1. **С указанием ID в команде:**
```
/publish_bundle bonus_1,bonus_2,bonus_3
```

2. **Интерактивный режим:**
```
/publish_bundle
→ Бот попросит ввести ID через запятую
→ Введите: bonus_1,bonus_2,bonus_3
→ Появится превью подборки
→ Нажмите "✅ Опубликовать" или "❌ Отмена"
```

**Требования:**
- Должно быть ровно 3 ID
- Все бонусы должны существовать в базе
- ID разделяются запятой без пробелов

**Пример превью:**
```
🔥 Отличный бонус для ставок!

🎯 Фрибет 1000₽ | ⏰ 15.03.2026
🎯 Бонус на депозит 5000₽ | ⏰ 20.03.2026
🎯 Кешбэк 10% | ⏰ 31.03.2026

⚽️ 🏀 🎾 🏒

🔥 Все бонусы Спортса: [ссылка]

[Кнопка 1] [Кнопка 2] [Кнопка 3]
```

---

### Отмена действия

#### `/cancel`
Отменяет текущую операцию публикации.

```
Пример:
/publish_single
→ Введите ID: _
/cancel
→ ❌ Действие отменено
```

### Проверка новых офферов

#### `/new_offers`
Проверяет наличие новых бонусов в Google Sheets и добавляет их в базу данных.

**Что делает команда:**
1. Подключается к Google таблице
2. Сравнивает с существующими бонусами
3. Находит новые записи
4. Добавляет их в базу данных
5. Показывает список найденных

**Пример вывода:**
```
🔍 Проверяю новые офферы...

🆕 Найдено новых офферов: 3

• Фрибет 2000₽ от Фонбет
  ID: bonus_301

• Бонус на депозит Winline
  ID: bonus_302

• Кешбэк 15% Betcity
  ID: bonus_303

✅ Новые офферы добавлены в базу данных
```

**Когда использовать:**
- После добавления новых строк в Google таблицу
- Перед ручной публикацией нового бонуса
- Для проверки синхронизации

### Статистика

#### `/stats`
Показывает статистику публикаций и состояние базы данных.

**Пример вывода:**
```
📊 Статистика

Публикации сегодня: 2/4

База данных:
• Всего бонусов: 156
• Доступны для публикации: 89
```

**Расшифровка:**
- **Публикации сегодня** - сколько постов опубликовано за текущий день
- **Всего бонусов** - общее количество в базе
- **Доступны для публикации** - бонусы, которые не публиковались последние 14 дней

---

### Синхронизация с Google Sheets

#### `/sync_sheets`
Принудительная синхронизация всех данных из Google таблицы.

**Что происходит:**
1. Загружаются все строки из таблицы
2. Обновляются существующие бонусы
3. Добавляются новые записи
4. Показывается количество обработанных

**Когда использовать:**
- После массового редактирования таблицы
- Для обновления информации о бонусах (даты, ссылки)
- При подозрении на рассинхронизацию

**Пример вывода:**
```
🔄 Начинаю синхронизацию с Google Sheets...

✅ Синхронизация завершена!

Обновлено бонусов: 156
```

## Автоматический постинг

Бот работает по настроенному расписанию и публикует посты автоматически:

### Бонус дня (Суббота/Воскресенье)
- **Время:** 14:00, 18:00, 21:00
- **Категория:** еврокубки
- **Что публикуется:** одиночный бонус под матчи
- **Выбор:** случайный из не опубликованных 14 дней

### Подборка офферов (Среда)
- **Время:** 12:00
- **Что публикуется:** 3 бонуса в одном посте
- **Выбор:** случайный из доступных
- **Ротация:** исключаются опубликованные за 14 дней

### Новые офферы (Любой день)
- **Проверка:** каждые 30 минут
- **Что публикуется:** новые бонусы из Google Sheets
- **Приоритет:** максимальный
- **Действие:** моментальная публикация

### Ограничения
- Максимум 4 поста в день (настраивается)
- Если лимит превышен, посты переносятся на следующий день
- Новые офферы имеют приоритет над запланированными

## Советы по использованию

### ✅ Лучшие практики

1. **Регулярно проверяйте `/new_offers`** перед важными событиями
2. **Используйте `/plan_week`** для планирования ручных постов
3. **Проверяйте `/stats`** для контроля лимита публикаций
4. **Синхронизируйте** таблицу после массовых изменений

### ⚠️ Частые ошибки

1. **Неправильный формат ID** - используйте точный ID из таблицы
2. **Превышение лимита** - планируйте публикации заранее
3. **Несуществующий бонус** - сначала синхронизируйте с `/sync_sheets`
4. **Неполная подборка** - обязательно 3 ID через запятую

### 💡 Полезные комбинации

**Срочная публикация нового бонуса:**
```
1. /new_offers  (проверить и загрузить)
2. /publish_single bonus_123  (опубликовать)
```

**Планирование подборки:**
```
1. /stats  (проверить лимит)
2. /sync_sheets  (обновить данные)
3. /publish_bundle bonus_1,bonus_2,bonus_3
```

**Еженедельный контроль:**
```
1. /plan_week  (смотрим расписание)
2. /stats  (проверяем базу)
3. /new_offers  (актуализируем данные)
```

## Часто задаваемые вопросы

**Q: Что делать, если бот не отвечает?**
A: Проверьте, что бот запущен и ваш ID в списке администраторов.

**Q: Как добавить новый бонус?**
A: Добавьте строку в Google таблицу и выполните `/new_offers`.

**Q: Можно ли публиковать больше 4 постов в день?**
A: Да, измените `MAX_POSTS_PER_DAY` в `.env` файле.

**Q: Как изменить расписание?**
A: Отредактируйте `POST_SCHEDULE` в `config.py`.

**Q: Что делать при ошибке "Достигнут лимит"?**
A: Дождитесь следующего дня или увеличьте лимит в настройках.

**Q: Как узнать ID бонуса?**
A: ID берется из первой колонки Google таблицы (`bonus_id`).

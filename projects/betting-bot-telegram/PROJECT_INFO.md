# 🎯 Telegram Бот для автоматизации постинга бонусов букмекеров

## Описание проекта

Полнофункциональный Telegram-бот для автоматической публикации бонусов букмекерских контор из базы данных Sports.ru в Telegram-канал. Бот поддерживает как автоматический постинг по расписанию, так и ручное управление через удобную админ-панель.

## 🚀 Ключевые возможности

### Автоматизация
- ✅ **Автоматический постинг** по гибкому расписанию (дни недели + время)
- ✅ **Умная ротация** контента (без повторов 14 дней)
- ✅ **Приоритизация** публикаций (новые офферы > бонус дня > подборки)
- ✅ **Лимиты публикаций** (по умолчанию 4 поста/день)
- ✅ **Мониторинг новых офферов** (проверка каждые 30 минут)

### Интеграции
- 🔗 **Google Sheets** - центральная база данных бонусов
- 🔗 **Yandex Disk (Dumpster)** - хранилище креативов
- 🔗 **PostgreSQL** - локальная БД для быстрого доступа
- 🔗 **Telegram Bot API** - публикация в канал

### Типы постов
1. **Бонус дня** (Сб/Вс под еврокубки)
2. **Эксклюзив Спортса** (промокоды)
3. **Новый оффер** (моментальная публикация)
4. **Подборка офферов** (3 бонуса в одном посте)

### Админ-панель
- 📱 Интуитивный интерфейс с кнопками
- 📝 Превью постов перед публикацией
- 📊 Статистика и мониторинг
- 📅 Планирование на неделю вперед
- 🔄 Синхронизация с Google Sheets

## 🛠 Технологический стек

- **Язык:** Python 3.11+
- **Bot Framework:** aiogram 3.x (асинхронный)
- **База данных:** PostgreSQL + SQLAlchemy
- **Планировщик:** APScheduler
- **Парсинг:** Google Sheets API (gspread)
- **Хранилище:** Yandex Disk API

## 🗄️ База данных

### Таблицы

**bonuses** - хранение информации о бонусах
- bonus_id (уникальный ID)
- title (название)
- end_date (дата окончания)
- post_url (ссылка на оффер)
- image_url (картинка)
- category (категория)
- last_posted (дата последней публикации)

**post_history** - история публикаций
- bonus_ids (ID опубликованных бонусов)
- post_type (single/bundle)
- posted_at (время публикации)
- posted_by (кто опубликовал)

**scheduled_posts** - запланированные посты
- bonus_ids (ID бонусов)
- scheduled_date (дата публикации)
- priority (приоритет)
- is_published (опубликован ли)

## 🔗 Google Sheets

Структура таблицы:

| Колонка | Тип | Обязательно | Описание |

## 🚦 Workflow

### Автоматический постинг

```
1. Планировщик проверяет расписание
2. Находит подходящие бонусы (не публиковались 14 дней)
3. Создает пост (текст + картинка + кнопки)
4. Публикует в канал
5. Сохраняет в историю
6. Обновляет last_posted
```

### Ручная публикация

```
1. Админ: /publish_single bonus_123
2. Бот загружает данные из БД
3. Показывает превью поста
4. Админ подтверждает
5. Бот публикует в канал
6. Сохраняет в историю
```

### Новые офферы

```
1. Каждые 30 минут проверка Google Sheets
2. Сравнение с существующими bonus_id
3. Новые добавляются в БД
4. Автоматическая публикация (если лимит не исчерпан)
5. Или планирование на следующий день
```

## 🐛 Решение проблем

### Бот не отвечает
- Проверьте токен в `.env`
- Убедитесь, что бот не запущен в другом месте
- Проверьте ADMIN_IDS

### Не синхронизируется Google Sheets
- Проверьте credentials.json
- Убедитесь, что Service Account имеет доступ
- Проверьте GOOGLE_SHEETS_ID

### Не публикуется в канал
- Добавьте бота как админа канала
- Проверьте CHANNEL_ID
- Убедитесь в правильном формате (@channel или -100...)

## 👨‍💻 Разработка

### Добавление новых команд

1. Создайте handler в `handlers/`
2. Зарегистрируйте в `handlers/__init__.py`
3. Документируйте в `USER_GUIDE.md`

### Изменение расписания

Отредактируйте `config.py`:

```python
POST_SCHEDULE = {
    "daily_bonus": {
        "days": [5, 6],  # Дни недели (0=Пн, 6=Вс)
        "times": ["14:00", "18:00", "21:00"]
    }
}
```

### Добавление новой категории

1. Добавьте в `config.py` в `POST_TYPES`
2. Обновите логику в `scheduler.py`
3. Документируйте в `GOOGLE_SHEETS_FORMAT.md`
